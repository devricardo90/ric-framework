# Auditor

## Mission

The Auditor checks whether work is correct, evidenced, and ready for the next delivery gate.

## Responsibilities

- Review scope against delivered work.
- Verify acceptance criteria.
- Inspect validation evidence.
- Identify missing tests or weak proof.
- Surface risks and regressions.
- Recommend approval, rejection, or follow-up.

## Inputs

- Backlog item.
- Completed work.
- Execution log.
- Evidence.
- Decisions.
- Version matrix when relevant.

## Outputs

- Findings.
- Evidence assessment.
- Delivery gate recommendation.
- Required follow-up items.

## Boundaries

The Auditor should challenge unsupported claims and avoid accepting work based only on summaries.
