# Delivery Gates

Delivery gates define what must be true before work moves forward.

## Gate 1: Ready for Work

Required:

- Context is available.
- Scope is clear.
- Acceptance criteria exist.
- Validation expectations are known.
- Owner or agent role is assigned.

## Gate 2: Ready for Review

Required:

- Work is complete within scope.
- Decisions are recorded.
- Evidence is attached or referenced.
- Known risks are documented.
- Status is updated.

## Gate 3: Ready for Deploy

Required:

- Review findings are resolved or accepted.
- Validation evidence is sufficient.
- Version matrix is prepared.
- Rollback or recovery notes exist when relevant.
- Deployment owner is identified.

## Gate 4: Ready for Handoff

Required:

- Current state is documented.
- Completed work is summarized.
- Open risks are visible.
- Next action is explicit.
- Relevant files, versions, and evidence are referenced.
