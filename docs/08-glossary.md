# Glossary

## RIC Framework

The documentation-first, stack-agnostic execution framework defined in this repository.

## Protocolo RIC

The same framework name expressed as a protocol name. Use consistently with RIC Framework.

## Backlog

The governed list of proposed, active, blocked, and completed work.

## Status

The current project state, including active task, owner, blockers, risks, and next action.

## Decision

A recorded choice that affects project direction, implementation, delivery, or maintenance.

## Evidence

Proof that work was validated. Evidence can include command output, screenshots, logs, review notes, or manual verification.

## Version Matrix

A record mapping versions, commits, environments, validation status, and deployment notes.

## Session Handoff

A concise document that lets the next contributor resume work without guessing.

## Orchestrator

The role responsible for planning, sequencing, delegation, status, and handoff.

## Executor

The role responsible for completing scoped work and returning evidence.

## Auditor

The role responsible for reviewing work, evidence, risks, and delivery readiness.
