# Principles

## Documentation First

Work starts from written context and ends with written evidence. Chat history can support execution, but it is not the system of record.

## Stack Agnostic

RIC Framework applies to software, documentation, automation, operations, and research projects without requiring a specific toolchain.

## Evidence Before Completion

A task is not done because an agent or contributor says it is done. Completion requires validation evidence appropriate to the work.

## Version Discipline

Versions, commits, releases, environments, migrations, and validation status must be traceable.

## Controlled Agent Orchestration

Agents need roles, scope, inputs, outputs, and review expectations. Delegation without boundaries creates risk.

## Handoff Is Required Work

Stopping work without handoff creates hidden cost. Every meaningful session should leave the next contributor with current state and next action.

## Backlog Governance

The backlog is not a dumping ground. Items should have scope, priority, acceptance criteria, and status.

## Practical Minimums

RIC Framework favors useful records over ceremony. Keep documents short enough to maintain and clear enough to execute.
