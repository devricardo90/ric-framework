# Project Context

## Project Name

Sample RIC Project

## Purpose

Demonstrate how a project using RIC Framework organizes context, status, backlog, decisions, and handoff records.

## Target Users

- Project owner.
- Orchestrator.
- Executor.
- Auditor.

## Success Criteria

- A new contributor can understand the project state from the documents.
- Backlog items include acceptance criteria and validation expectations.
- Handoff notes identify the next action.

## Scope

## In Scope

- Documentation structure.
- Example governance records.
- Sample execution state.

## Out of Scope

- Product code.
- Dependencies.
- Deployment automation.

## Constraints

- Keep all records stack-agnostic.
- Use Markdown only.

## Environments

| Environment | Purpose | URL or Reference | Owner |
| --- | --- | --- | --- |
| Local docs | Documentation review | Repository files | Project owner |

## Risks

| Risk | Impact | Mitigation |
| --- | --- | --- |
| Documents become stale | Contributors may act on wrong state | Update status and handoff at the end of each session |

## Working Assumptions

- The project is in initialization stage.
- No implementation stack has been selected.
