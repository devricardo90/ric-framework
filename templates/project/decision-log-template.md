## Purpose

This log records all approved architecture, stack, provider, version, deployment, and lifecycle decisions for this project.

A decision must be recorded here before execution depends on it. A recommendation is not an approval.

Any change to an approved decision requires Trigger authorization and a new entry.

---

## Decision Format

Each entry must include:

```
## [DECISION-ID] — [Title]

Date:        [YYYY-MM-DD]
Area:        [Stack / Provider / Database / Authentication / Storage / Deployment / Architecture / Lifecycle]
Status:      [Approved / Superseded by DECISION-ID / Pending]
Supersedes:  [DECISION-ID or None]

### Decision

[State the approved choice in one or two sentences.]

### Reason

[Why was this decision made? Connect it to product requirements, MVP constraints, or known risks.]

### Options Considered

- [Option 1]: [Brief tradeoff note]
- [Option 2]: [Brief tradeoff note]

### Impact

[Expected effect on execution, validation, maintenance, security, deploy, or handoff.]

### Reversal Cost

[What it would cost to change this decision later, when known.]

### Trigger Approval

[Date and summary of Trigger approval, or "Pending"]
```
