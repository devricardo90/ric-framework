# Agent Execution

Use this playbook when assigning work to an AI agent.

## Steps

1. Select the agent role.
2. Create or update an `agent-brief.md`.
3. Provide project context, active backlog item, relevant decisions, and validation expectations.
4. Define included and excluded scope.
5. Require outputs, evidence, risks, and handoff notes.
6. Review the agent output before accepting completion.
7. Use the Auditor role for high-risk work.

## Agent Output Requirements

- Work completed.
- Files or artifacts changed.
- Validation evidence.
- Risks or gaps.
- Recommended next action.

## Acceptance Rule

Agent work is accepted only when it satisfies scope, evidence, and handoff requirements.
