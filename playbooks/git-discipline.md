# Git Discipline

Use this playbook to keep version history useful.

## Before Starting

1. Check repository status.
2. Identify existing uncommitted work.
3. Confirm the intended backlog item.
4. Avoid mixing unrelated changes.

## During Work

1. Keep changes scoped.
2. Record decisions that affect future work.
3. Capture validation evidence before committing.

## Before Commit

1. Review changed files.
2. Confirm no dependencies, generated files, or secrets were added accidentally.
3. Run relevant validation.
4. Update status and handoff documents when needed.

## Commit Message Format

Use concise messages with a type and scope when useful.

Examples:

- `docs: initialize ric framework structure`
- `fix: correct release validation notes`
- `test: add deployment checklist coverage`
