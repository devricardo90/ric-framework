# Close Task

Use this playbook before marking a backlog item as done.

## Steps

1. Confirm the task stayed within scope.
2. Check every acceptance criterion.
3. Record validation evidence.
4. Record decisions made during the task.
5. Update `backlog.md`.
6. Update `STATUS.md`.
7. Complete a done record using `done-template.md` when the work is significant.
8. Update `session-handoff.md` if work is pausing or changing owners.

## Done Requirements

- Acceptance criteria are satisfied.
- Evidence is discoverable.
- Risks are documented.
- Next action is clear.
